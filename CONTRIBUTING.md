# Contributing

Thank you for your interest in this project.

## Status

This is a draft technical review package, not a production release.
Contribution guidelines below apply once the repository is public.

## Reporting issues

- For bugs in the verifier or test vectors, open a GitHub issue.
- For security issues, see SECURITY.md. Do not open public issues for
  security findings.
- For protocol design questions or proposed protocol changes, open a
  GitHub discussion before submitting a pull request.

## Pull requests

- Fork the repository and create a feature branch from `main`.
- Ensure `pytest` and the Go canonical check both pass before
  submitting.
- Match the existing code style. Run any linters added to CI.
- Include a clear description of the change and motivation.
- Cryptographic changes require additional review and may take longer
  to merge.
